#### Test Git project on GitHub

Выкладываем репозиторий на GitHub  
************************************

Чтобы выложить свой локальный Git-репозиторий на GitHub, выполните следующие шаги:

1. **Создайте аккаунт на GitHub, если у Вас его ещё нет.**

2. **Создайте новый репозиторий на GitHub:**
- Перейдите на сайт GitHub и войдите в свой аккаунт.
- Нажмите кнопку «New» или «Создать новый репозиторий» (в зависимости от интерфейса).
- Заполните необходимую информацию о репозитории, такую как название, описание и выбор между публичным и приватным.
- Не забудьте отметить галочкой пункт «Initialize this repository with a README», если вы хотите создать репозиторий с начальным файлом README.

3. **Получите URL вашего удалённого репозитория на GitHub.**  
После создания репозитория GitHub предоставит вам URL-адрес вашего нового удалённого репозитория. 

5. **Свяжите ваш локальный репозиторий с удалённым на GitHub:**

* Откройте терминал или командную строку и перейдите в каталог вашего локального репозитория.
* Выполните команду `git remote add origin <your_github_repo_url>`, заменив шаблон URL-адреса на фактический URL своего удалённого репозитория.

5. **Отправьте изменения из локального репозитория в удалённый на GitHub:**

+ Убедитесь, что все ваши изменения зафиксированы в локальном репозитории с помощью команд git add . и git commit
+ Отправьте изменения в удалённый репозиторий с помощью команды `git push -u origin master`, если Ваша основная ветка называется ***master***.

6. **Проверьте, что изменения были успешно отправлены на GitHub:**

- Перейдите на страницу своего репозитория на GitHub и убедитесь, что файлы и изменения там появились.

Готово! Теперь Ваш локальный Git-репозиторий связан с удалённым репозиторием на GitHub, и Вы можете продолжать работать над проектом, отправляя изменения на GitHub по мере необходимости.
